Expense Tracker with Data Visualization & AI Insights

A full-stack MERN application for tracking expenses, visualizing spending, and receiving AI-powered financial insights.

Project Structure Overview
  • backend/: Node.js + Express REST API, MongoDB, JWT Auth, AI integration
  • frontend/: React (Vite), Tailwind CSS, Data Visualization, Auth, Dashboard
Backend (Node.js + Express)
Authentication & User Management
  • Endpoints: /api/auth/register and /api/auth/login
  • Controller: authController.js handles registration, login, JWT token generation, and welcome emails.
  • Model: User.js (Mongoose) with password hashing and validation.
  • Middleware: authMiddleware.js protects private routes using JWT.
Middleware
  • authMiddleware.js: Checks for a valid JWT token in the Authorization header. If valid, attaches the user to the request; otherwise, blocks access to protected routes.
  • Middleware is used in expenseRoutes.js to ensure only authenticated users can access, add, update, or delete expenses and get AI insights.
  • Other middleware includes express.json() for parsing JSON bodies and cors() for enabling cross-origin requests.
Expense Management
  • Endpoints: /api/expenses (CRUD) and /api/expenses/insights (AI insights)
  • Controller: expenseController.js for CRUD, category breakdown, and AI-powered insights (Google Gemini API).
  • Model: Expense.js (Mongoose) with user reference, amount, category, date, and description.
Routing
  • authRoutes.js: Handles /register and /login POST requests.
  • expenseRoutes.js: Handles expense CRUD and insights, all protected by JWT middleware.
AI Integration
  • Uses Google Gemini API to analyze user expenses and generate personalized financial insights.
  • Endpoint: GET /api/expenses/insights (requires authentication).
Frontend (React + Vite)
Authentication Flow
  • Login.jsx: User login form, calls backend /api/auth/login.
  • Register.jsx: User registration form, calls backend /api/auth/register.
  • Auth state managed via AuthContext.jsx and JWT in localStorage.
  • Protected routes via ProtectedRoute.jsx.
Dashboard & Data Visualization
  • Dashboard.jsx: Main page after login. Shows summary, charts, and AI insights.
  • Charts: CategoryPieChart.jsx, MonthlyBarChart.jsx, WeeklyLineChart.jsx for visualizing spending.
  • Expense management: ExpenseForm.jsx (add), ExpenseList.jsx (list/delete), ExpenseFilter.jsx (filter).
  • AI Insights: Button in dashboard fetches analysis from backend and displays in a modal.
Routing & Navigation
  • React Router is used for navigation between login, register, and dashboard pages.
  • ProtectedRoute ensures only authenticated users can access the dashboard.
How It Works
  1. User registers or logs in. JWT token is stored in browser.
  2. Authenticated requests to backend include JWT in Authorization header.
  3. Users can add, view, update, and delete expenses. All data is user-specific.
  4. Dashboard visualizes expenses by category, month, and week using interactive charts.
  5. Clicking "Get AI Insights" sends expenses to backend, which uses Gemini AI to generate a personalized analysis.
Key Features
  • Modern UI with glassmorphism and animated backgrounds (Tailwind CSS).
  • Secure authentication with JWT and password hashing.
  • Full CRUD for expenses, with filtering and sorting.
  • Interactive charts for data visualization.
  • AI-powered financial insights (Google Gemini API).
  • Responsive and mobile-friendly design.
Live Demo

To experience the app:

  1. Start the backend server (node backend/server.js).
  2. Start the frontend (npm run dev in frontend/).
  3. Register a new account and explore the dashboard.

(Note: You need MongoDB and a Gemini API key for full functionality.)
